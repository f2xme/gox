# gox - Go eXtended utilities

## Project Overview
A collection of wrapped Go third-party libraries providing unified API style and out-of-the-box configuration.
Focuses on improving development efficiency and code consistency. Each package is built on top of mature open-source libraries.

Repository: github.com/f2xme/gox

## Architecture
- Multi-package utility library, each top-level directory is an independent package
- Packages are independent and do not depend on each other
- Uses adapter pattern to wrap third-party libraries
- Unified error handling and configuration style

## Package List

### encrypt - Encryption Tools
- AES, RSA, Hash and other encryption algorithms

### errorx - Enhanced Error Handling
- Structured error handling
- Error chain tracing

### graceful - Graceful Shutdown
- Application lifecycle management
- Signal handling and resource cleanup

### httpx - HTTP Utilities
- adapter/ginadapter - Gin framework adapter

### idgen - ID Generator

- HMAC, RSA, ECDSA signing algorithms
- Token generation, validation, revocation

### logx - Logging Wrapper
- adapter/zapadapter - Zap logger adapter
- Structured logging and context propagation

### metrics - Metrics Monitoring
- Prometheus metrics wrapper
- Counter, gauge, histogram support

### email - Email Services

### sms - SMS Services

### oss - Object Storage

### pager - Pagination Utilities

### payment - Payment Services

### queue - Queue Wrapper
- adapter/memadapter - In-memory queue adapter

### ratelimit - Rate Limiting

### validator - Data Validation
- Struct validation wrapper
- Custom validation rules support

## Code Conventions
- Exported functions and types must have doc comments
- Errors should be returned, not panicked. Use fmt.Errorf with %w for wrapping
- Packages should be small and focused on a single concern
- Packages remain independent, no cross-imports within library
- Table-driven tests are preferred

## Build & Test
```bash
go build ./...           # Build all packages
go test ./...            # Run all tests
go test -v ./...         # Verbose output
go test -race ./...      # Race detection
gofmt -w .               # Format code
```

## Import Usage
```go
import "github.com/f2xme/gox/<package>"
import "github.com/f2xme/gox/<package>/adapter/<adapter>"
```

## Design Patterns
- Adapter Pattern: Unified wrapping of third-party libraries
- Factory Pattern: New/MustNew constructor functions
- Options Pattern: Optional configuration parameters
- Interface Segregation: Minimal interface dependencies
